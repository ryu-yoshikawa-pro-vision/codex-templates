# ADR 0001: PLAN/TASKS/REPORT運用の厳格化

## Status
- Accepted

## Context
- Codexが多様なタスクを扱う際、計画・タスク・行動ログが散逸しやすい。
- セッション内で複数のタスクが発生するため、追記運用とログの一貫性が必要。

## Decision
- PLANに「Thinking Log」を設け、思考/判断理由の記録先を統一する。
- TASKSに「Discovered」追記運用を明文化する。
- REPORTに「行動ログ必須」「Progress行」を追加する。
- AGENTS.mdでこれらの記録先と運用ルールを強制する。

## Consequences
- 記録の粒度が上がり、追跡性が向上する。
- 反面、テンプレートとログの記述量が増える。
