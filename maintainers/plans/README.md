# maintainers/plans Usage

## Purpose
- Store source-repo planning documents and plan-to-implementation handoff records.

## Naming Rule
- Create plan files as: `{yyyy-mm-dd}_{HHMMSS}_{plan_name}.md`
- Use JST (`Asia/Tokyo`) for the date/time portion.

## Recommended Workflow
1. Copy `maintainers/plans/TEMPLATE.md` to a new plan file.
2. Record source-repo goal, consumer-facing impact, spec impact, tasks, risks, and validation.
3. Keep the plan current while implementation is in progress.
